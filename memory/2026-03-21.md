## 10:47 — Vision Brainstorm (Rob)
Rob laid out a fundamentally bigger vision than "YouTube scripts":

### Core Insight
The app should **encode virality knowledge into the creation process itself**. Grace shouldn't need to know what an "Iceberg Hook" is. She just sees better options. Like CapCut — templates that encode good principles, you just pick one.

### Key Decisions
1. **Outline-first for YouTube** — ONE outline that user edits. Show 2-3 structural approaches to pick from.
2. **Block builder with tap-to-swap** — generated ON DEMAND (not pre-generated). Keep copy of previous block so user can compare and pick.
3. **Reels: AI-first** since they're short. Generate full script, then block builder to interchange elements.
4. **All formats share the pattern**: simple input → visual structure → tap-to-swap → progressive detail
5. **Non-video content**: image + text overlay compositing. User can choose overlay style. Must be editable (user can change text after generation).
6. **Image+text design should encode best practices**: where to place text, font choices, contrast rules — all from KB.
7. **Topic generation**: AI suggests topics for Grace. Must track: days since last content type, days since topic was discussed, ensure even spacing and no recycling.

- Rob wants composited approach (text baked onto image, like Canva)
- But needs to be EDITABLE after — text overlay as a separate layer the user can modify
- Rob asked: can we work through Canva API? If not, need our own text overlay editor.
